# Claude Code チュートリアルまとめ

## 基本操作

- やりたいことを日本語でそのまま伝えるだけでOK
- 雑な伝え方でも大丈夫（「なんかエラー出た」等）

## よく使うコマンド

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプを表示 |
| `/clear` | 会話を完全リセット（元に戻せない） |
| `/compact` | 会話を要約して圧縮（文脈は残る） |

- 迷ったら `/compact` が安全

## 許可メッセージ（英語→日本語）

| 英語 | 意味 |
|------|------|
| Allow Read | ファイルを読む |
| Allow Write | ファイルを作成/上書き |
| Allow Edit | ファイルを編集 |
| Allow Bash | コマンドを実行 |
| Allow Glob | ファイルを検索 |
| Allow Grep | ファイルの中身を検索 |
| Allow | 今回だけ許可 |
| Allow always | 今後ずっと許可 |
| Deny | 拒否 |

## CLAUDE.md

- プロジェクトのルールを書く設定ファイル
- 自動作成されない。自分で作る
- 置く場所:
  - プロジェクトのルート → そのプロジェクト専用
  - `~/.claude/CLAUDE.md` → 全プロジェクト共通
- `/clear` しても CLAUDE.md の内容は引き継がれる

## 周辺用語

| 用語 | 意味 |
|------|------|
| MCP | 追加機能をつなげる仕組み（スマホにアプリを入れるイメージ） |
| Skills | ショートカットコマンド（`/commit` 等） |
| Hooks | 特定のタイミングで自動実行されるスクリプト（上級者向け） |
| コンテキストウィンドウ | AIが一度に覚えていられる量 |
| プロンプト | AIへの指示や質問 |
| トークン | AIが処理するテキストの単位。多い＝コスト高い |

## Git の基本

- **Git** = 自分のPC上でセーブポイントを管理するツール
- **GitHub** = セーブデータをネット上に保管するサービス

### 用語

| 用語 | 意味 |
|------|------|
| リポジトリ (repo) | Gitで管理されたフォルダ |
| コミット (commit) | セーブポイントを作る |
| プッシュ (push) | GitHubにアップロード |
| プル (pull) | GitHubからダウンロード |
| ブランチ (branch) | 本体に影響を与えず別の作業をする枝分かれ |

### 基本コマンド

```
git init                    → リポジトリ作成（最初に1回だけ）
git add ファイル名            → セーブ対象に追加
git commit -m "メッセージ"    → セーブポイント作成
git diff                    → 何が変わったか確認
git checkout -- ファイル名    → 変更を元に戻す
```

Claude Code では「コミットして」と言うだけで自動でやってくれる。

## VSCode との連携

- 会話履歴はセッション限り。フォルダを開き直すと消える（コードとGitは残る）
- **コード選択→質問**: エディタでコードを選択 → チャットに「これ何？」と聞くだけ
- **ソース管理**: 左の枝分かれアイコンで、変更箇所を赤（削除）/緑（追加）で確認できる
- **ターミナル**: `Ctrl + @` で開く（Claude Codeがあれば普段は不要）

## 学習のコツ

- 一度に全部やらなくてOK。1日1〜2項目ずつ進める
- わからないことはClaude Codeに「〇〇って何？」と聞くだけでいい
- 失敗しても大丈夫。Gitがあれば元に戻せる
- 実際に何か作りたいものを決めると上達が早い
