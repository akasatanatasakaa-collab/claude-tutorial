基本操作

  • やりたいことを日本語でそのまま伝えるだけでOK
  • 雑な伝え方でも大丈夫（「なんかエラー出た」等）

よく使うコマンド

コマンド説明
/helpヘルプを表示
/clear会話を完全リセット（元に戻せない）
/compact会話を要約して圧縮（文脈は残る）
迷ったら /compact が安全

許可メッセージ（英語→日本語）

英語意味
Allow Readファイルを読む
Allow Writeファイルを作成/上書き
Allow Editファイルを編集
Allow Bashコマンドを実行
Allow Globファイルを検索
Allow Grepファイルの中身を検索
Allow今回だけ許可
Allow always今後ずっと許可
Deny拒否

CLAUDE.md

  • プロジェクトのルールを書く設定ファイル
  • 自動作成されない。自分で作る
  • 置く場所:
    • プロジェクトのルート → そのプロジェクト専用
    • ~/.claude/CLAUDE.md → 全プロジェクト共通
  • /clear しても CLAUDE.md の内容は引き継がれる

周辺用語

用語意味
MCP追加機能をつなげる仕組み（スマホにアプリを入れるイメージ）
Skillsショートカットコマンド（/commit 等）
Hooks特定のタイミングで自動実行されるスクリプト（上級者向け）
コンテキストウィンドウAIが一度に覚えていられる量
プロンプトAIへの指示や質問
トークンAIが処理するテキストの単位。多い＝コスト高い

Git の基本

  • Git = 自分のPC上でセーブポイントを管理するツール
  • GitHub = セーブデータをネット上に保管するサービス

Git 用語

用語意味
リポジトリ (repo)Gitで管理されたフォルダ
コミット (commit)セーブポイントを作る
プッシュ (push)GitHubにアップロード
プル (pull)GitHubからダウンロード
ブランチ (branch)本体に影響を与えず別の作業をする枝分かれ
マージ (merge)ブランチの変更を本体に合流させる

Git コマンド

git init                    → リポジトリ作成（最初に1回だけ）
git add ファイル名            → セーブ対象に追加
git commit -m "メッセージ"    → セーブポイント作成
git diff                    → 何が変わったか確認
git checkout -- ファイル名    → 変更を元に戻す
git checkout -b ブランチ名   → 新しいブランチを作って移動
git merge ブランチ名          → ブランチを本体に合流
Claude Code では「コミットして」と言うだけで自動でやってくれる

VSCode との連携

機能やり方使う場面
コード選択→質問エディタで選択→チャットに質問「ここ何？」「ここ変えて」
ソース管理左の枝分かれアイコン変更箇所を視覚的に確認
ターミナルCtrl + @Claude Codeがあれば普段は不要
会話履歴はセッション限り。フォルダを開き直すと消える（コードとGitは残る）

PR（プルリクエスト）

  • PR = 「この変更を本体に取り込んでいいですか？」という提案
  • レビュー = 提案された変更をチェックすること
  • マージ = 承認されたPRを本体に取り込むこと
git checkout -b ブランチ名   → 作業用ブランチを作る
（コードを変更してコミット）
git push -u origin ブランチ名 → GitHubに送る
gh pr create                  → PRを作成
gh pr merge                   → マージ（本体に取り込む）

プロンプトの書き方のコツ

レベル例
雑でもOK「リンク追加して」
普通「index.htmlにクイズへのリンクを追加して」
上手い「index.htmlの花カウンターの下に、quiz/index.htmlへのリンク集を追加して。ダークモードに合うデザインで」
完璧を目指さなくていい。結果を見て「ここ変えて」と追加指示すればOK

  • 簡単なWebページを一から作ってもらう

  • 既存のコードを説明してもらう

  • クイズアプリを一から作る
  • 複数ファイルのプロジェクトを扱う（quiz/フォルダ）
  • テストコードを書いてもらう

  • 「コミットして」だけでGit操作を任せる
完了 レベル5: 応用テクニック
  • MCP サーバーを追加して機能を拡張する
  • Skills（カスタムコマンド）を自作する
  • Hooks で自動化する
  • 大きなコードベースを読み解いてもらう
  • PRの作成・レビューをやってもらう
進行中 レベル6: マスター
  • 複雑なアプリを設計から実装まで任せる
  • 既存のOSSを読み解いて改修する
  • CI/CDパイプラインを構築する（GitHub Pages）
  • チーム開発でClaude Codeを活用する
  • プロンプトの書き方を最適化する

学習のコツ

  • 一度に全部やらなくてOK。1日1〜2項目ずつ進める
  • わからないことはClaude Codeに「〇〇って何？」と聞くだけでいい
  • 失敗しても大丈夫。Gitがあれば元に戻せる
  • 実際に何か作りたいものを決めると上達が早い
